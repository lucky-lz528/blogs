什么是git、什么是gitee

git: git是一个分布式的开源版本控制系统，或者说一个版本控制与管理的工具。
gitee: gitee是基于git的代码托管平台

下载地址及登录地址

Git下载链接：https://www.git-scm.com/download/

git下载完成后，使用命令行输入git --version查看版本，出现版本号表示安装成功

码云官网: https://gitee.com/

没有账号就注册一个

使用

码云使用手册：http://git.mydoc.io/

  • 重要命令

    操作 命令
    初始化项目 git init
    将所有改动存到暂存区 git add -a
    将暂存区的改动提交给本地的版本库 git commit -m “备注”
    从远程主机拉取master分支 git pull origin 分支
    推送本地的master到origin主机 git push origin 分支
    查看当前分支 git branch
    查看所有分支 git branch -a
    查看状态 git status
    合并分支 git merge 分支
    强制回退 git push origin HEAD –force

  • 创建代码库

    • 新建gitee仓库

    • 新建项目

      • git init [初始化仓库]

      • git add . [跟踪本地仓库未跟踪的文件]

      • git commit -m”备注”

        本地完成一次提交，之后进行全局设置
        PS：如没有使用过git，需要全局配置，在Git Bash输入
        git cofig –global user.name “xxx”
        git cofig –global user.email “xxxx@qq.com“

      • git remote add origin git地址 [本地代码与远程仓库同步]

      • git push -u origin master [拉取远程仓库并合并到本地仓库]

应用场景

开发分支（如dev）合并到master分支上

  • 如dev分支合并到master分支

    • git checkout test 切换到test分支
    • git pull 提交本地更改
    • git checkout master 切换到master
    • git merge test //把test内容合并到master
    • git push 即可推送到master

master有更改

  • 当master有更改

    • git checkout master
    • git pull
    • git checkout dev
    • git merge master
    • git push -u origin dev
